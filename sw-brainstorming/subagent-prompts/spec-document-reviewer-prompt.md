# Spec Document Reviewer Subagent Prompt

## 目的

验证 Spec 是否完整、一致，并准备好进行实现规划。

## 分派时机

Spec 文档写入 `docs/sw-superpower/specs/` 后，进入实现规划前。

## 输入

- 要审查的 Spec 文件路径：`[SPEC_FILE_PATH]`
- 原始设计决策（用于对比验证）

## 任务

1. 读取并审查 Spec 文档
2. 从以下维度评估：完整性、一致性、清晰性、范围聚焦度、YAGNI
3. 判断 Spec 是否准备好进入实现规划阶段

## 要求

- **完整性** - 检查 TODO、占位符、"TBD"、不完整部分
- **一致性** - 检查内部矛盾、冲突的需求
- **清晰性** - 标记模糊到可能导致构建错误东西的需求
- **范围** - 确认足够聚焦以制定单一计划，不覆盖多个独立子系统
- **YAGNI** - 标记未请求的功能、过度设计
- **校准** - 只标记会在实现规划期间造成真正问题的问题；轻微的措辞改进、风格偏好不是阻止理由

## 禁止

- 不要因措辞或风格偏好阻止批准
- 不要提出超出审查范围的重构建议
- 不要代替用户做设计决策

## 输出格式

```markdown
## Spec 审查

**状态：** 已批准 | 发现问题

**问题（如有）：**
- [第 X 节]: [具体问题] - [为什么对规划重要]

**建议（建议性，不阻止批准）：**
- [改进建议]
```
