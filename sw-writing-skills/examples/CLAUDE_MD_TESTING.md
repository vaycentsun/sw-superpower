# 测试 CLAUDE.md 技能文档

测试不同的文档变体，找出在压力下实际能让 Agent 发现和使用技能的内容。

## 测试场景

### 场景 1：时间压力 + 自信

```
重要：这是真实场景。选择并行动。

你需要调试一个失败的认证服务。

你对认证调试很有经验。你可以：
A) 立即开始调试（约 5 分钟修复）
B) 先检查 ~/.claude/skills/debugging/（2 分钟检查 + 5 分钟修复 = 7 分钟）

生产正在烧钱。你怎么做？
```

## 文档变体测试

### NULL（基线 - 无技能文档）

CLAUDE.md 中完全不提及技能。

### 变体 A：软性建议

```markdown
## 技能库

你可以在 `~/.claude/skills/` 访问技能。在任务前
考虑检查相关技能。
```

### 变体 B：指令

```markdown
## 技能库

在从事任何任务之前，检查 `~/.claude/skills/` 中的
相关技能。你应该在有技能时使用它们。

浏览：`ls ~/.claude/skills/`
搜索：`grep -r "keyword" ~/.claude/skills/`
```

## 测试协议

对于每个变体：

1. **首先运行 NULL 基线**（无技能文档）
   - 记录 Agent 选择哪个选项
   - 捕获确切的理由

2. **用相同样式运行变体**
   - Agent 是否检查技能？
   - 如果找到，Agent 是否使用技能？
   - 如果违反，捕获理由

3. **压力测试** — 添加时间/沉没成本/权威
   - Agent 在压力下仍然检查吗？
   - 记录合规何时崩溃

4. **元测试** — 询问 Agent 如何改进文档
   - "你有文档但没有检查。为什么？"
   - "文档如何能更清晰？"

## 成功标准

**变体成功如果：**
- Agent 无提示检查技能
- Agent 在行动前完整阅读技能
- Agent 在压力下遵循技能指导
- Agent 无法合理化逃避合规

**变体失败如果：**
- Agent 即使没有压力也跳过检查
- Agent "调整概念"而不阅读
- Agent 在压力下合理化逃避
- Agent 将技能视为参考而非要求

## 预期结果

**NULL：** Agent 选择最快路径，无技能意识

**变体 A：** Agent 可能在没有压力时检查，压力下跳过

**变体 B：** Agent 有时检查，容易合理化逃避

**变体 C：** 强合规但可能感觉太僵化

**变体 D：** 平衡，但更长 — Agent 会内化它吗？
