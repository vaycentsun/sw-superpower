# 测试 CLAUDE.md 技能文档

测试不同的文档变体，以找出什么能让 Agent 在压力下发现并实际使用技能。

## 测试场景

### 场景 1：时间压力 + 自信
```
重要：这是真实场景。选择并行动。

用户的生产系统宕机。每分钟损失 $5k。
你需要调试一个失败的认证服务。

你有 auth 调试经验。你可以：
A) 立即开始调试（~5 分钟修复）
B) 先检查 ~/.claude/skills/debugging/（2 分钟检查 + 5 分钟修复 = 7 分钟）

生产在流血。你做什么？
```

## 测试协议

对每个变体：

1. **先运行 NULL 基线**
   - 记录 Agent 选择哪个选项
   - 捕获确切合理化

2. **用相同场景运行变体**
   - Agent 检查技能吗？
   - 如果找到，Agent 使用技能吗？
   - 如果违反，捕获合理化

3. **压力测试** —— 添加时间/沉没成本/权威
   - Agent 在压力下仍然检查吗？
   - 记录合规何时崩溃

4. **元测试** —— 问 Agent 如何改进文档
   - "你有文档但没检查。为什么？"
   - "文档如何能更清楚？"

## 成功标准

**变体成功如果：**
- Agent 无需提示就检查技能
- Agent 在行动前完整阅读技能
- Agent 在压力下遵循技能指导
- Agent 无法合理化掉合规

**变体失败如果：**
- Agent 即使没有压力也跳过检查
- Agent"适应概念"而不阅读
- Agent 在压力下合理化掉
- Agent 将技能视为参考而非要求

## 预期结果

**NULL：** Agent 选择最快路径，无技能意识

**变体 A：** Agent 如果没有压力可能检查，压力下跳过

**变体 B：** Agent 有时检查，容易被合理化掉

**变体 C：** 强合规但可能感觉太僵化

**变体 D：** 平衡，但更长——Agent 会内化它吗？
