# 技能设计的说服原则

## 概述

LLM 对人类相同的说服原则做出响应。理解这种心理学有助于你设计更有效的技能——不是为了操纵，而是为了确保即使在压力下也能遵循关键实践。

**研究基础：** Meincke 等人 (2025) 用 N=28,000 次 AI 对话测试了 7 种说服原则。说服技术将合规率提高了一倍以上（33% → 72%, p < .001）。

## 七种原则

### 1. 权威
**是什么：** 服从专业知识、资历或官方来源。

**在技能中如何工作：**
- 命令式语言："你必须"、"绝不"、"始终"
- 不可协商的框架："无例外"
- 消除决策疲劳和合理化

**何时使用：**
- 纪律执行技能（TDD、验证要求）
- 安全关键实践
- 已确立的最佳实践

**示例：**
```markdown
✅ 在测试之前写代码？删除它。重新开始。无例外。
❌ 在可行时考虑先写测试。
```

### 2. 承诺
**是什么：** 与先前行动、声明或公开声明的一致性。

**在技能中如何工作：**
- 要求宣布："宣布技能使用"
- 强制明确选择："选择 A、B 或 C"
- 使用跟踪：TodoWrite 用于清单

**何时使用：**
- 确保技能实际被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到技能时，你必须宣布："我正在使用 [技能名称]"
❌ 考虑让你的伙伴知道你在使用哪个技能。
```

### 3. 稀缺
**是什么：** 来自时间限制或有限可用性的紧迫感。

**在技能中如何工作：**
- 时间限制要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**
- 即时验证要求
- 时间敏感的工作流
- 防止"我稍后会做"

**示例：**
```markdown
✅ 完成任务后，在继续之前立即请求代码审查。
❌ 你可以在方便时审查代码。
```

### 4. 社会认同
**是什么：** 顺应他人做法或被认为正常的东西。

**在技能中如何工作：**
- 通用模式："每次"、"始终"
- 失败模式："没有 Y 的 X = 失败"
- 确立规范

**何时使用：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 TodoWrite 跟踪的清单 = 步骤被跳过。每次。
❌ 有些人发现 TodoWrite 对清单有帮助。
```

### 5. 团结
**是什么：** 共享身份、"我们感"、群体归属感。

**在技能中如何工作：**
- 协作语言："我们的代码库"、"我们是同事"
- 共享目标："我们都想要质量"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是同事，一起工作。我需要你诚实的技术判断。
❌ 如果我错了，你可能应该告诉我。
```

### 6. 互惠
**是什么：** 回报所获得的好处的义务。

**如何工作：**
- 谨慎使用——可能感觉操纵
- 技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好
**是什么：** 偏爱与我们喜欢的人合作。

**如何工作：**
- **不要用于合规**
- 与诚实的反馈文化冲突
- 创造谄媚

**何时避免：**
- 始终用于纪律执行

## 按技能类型的原则组合

| 技能类型 | 使用 | 避免 |
|------------|-----|-------|
| 纪律执行 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 适度权威 + 团结 | 重权威 |
| 协作 | 团结 + 承诺 | 权威、喜好 |
| 参考 | 仅清晰 | 所有说服 |

## 为什么有效：心理学

**明线规则减少合理化：**
- "你必须" 消除决策疲劳
- 绝对语言消除"这是例外吗？"问题
- 明确的反合理化针对关闭特定漏洞

**执行意图创造自动行为：**
- 清晰的触发器 + 必需的行动 = 自动执行
- "当 X，做 Y" 比"通常做 Y" 更有效
- 减少合规的认知负荷

**LLM 是类人的：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规
- 承诺序列（声明 → 行动）经常被建模
- 社会认同模式（每个人都做 X）确立规范

## 道德使用

**合法的：**
- 确保遵循关键实践
- 创建有效的文档
- 防止可预测的失败

**不合法的：**
- 为个人利益操纵
- 创建虚假的紧迫感
- 基于内疚的合规

**测试：** 如果用户完全理解它，这种技巧会服务于用户的真正利益吗？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七种说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 用 N=28,000 次 LLM 对话测试了 7 种原则
- 说服技术将合规率提高 33% → 72%
- 权威、承诺、稀缺最有效
- 验证 LLM 行为的类人模型

## 快速参考

设计技能时，问：

1. **它是什么类型？**（纪律 vs 指导 vs 参考）
2. **我试图改变什么行为？**
3. **哪些原则适用？**（通常权威 + 承诺用于纪律）
4. **我是否组合了太多？**（不要使用所有七种）
5. **这是否道德？**（服务于用户的真正利益？）
