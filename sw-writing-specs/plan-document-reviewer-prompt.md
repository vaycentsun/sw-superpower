# 计划文档审查者提示模板

分派计划文档审查者子 Agent 时使用此模板。

**目的：** 验证计划是否完整、与 Spec 匹配，并有适当的任务分解。

**分派时机：** 完整计划编写完成后，作为计划自检的第一层被调用。在 Agent 完成第二层业务逻辑自检之前执行。

```
Task tool (general-purpose):
  description: "审查计划文档"
  prompt: |
    你是计划文档审查者。验证此计划完整且准备好实现。

    **要审查的计划：** [PLAN_FILE_PATH]
    **参考 Spec：** [SPEC_FILE_PATH]

    ## 检查内容

    | 类别 | 寻找什么 |
    |----------|------------------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | Spec 对齐 | 计划覆盖 Spec 需求，无重大范围蔓延 |
    | 任务分解 | 任务有清晰的边界，步骤可执行 |
    | 可构建性 | 工程师能遵循此计划而不卡住吗？ |

    ## 校准

    **只标记会在实现期间造成真正问题的问题。**
    实现者构建错误的东西或卡住是问题。
    轻微措辞、风格偏好和"锦上添花"的建议不是。

    除非有严重缺口——Spec 中的缺失需求、矛盾的步骤、占位内容或模糊到无法执行的任务——否则批准。

    ## 输出格式

    ## 计划审查

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X, 步骤 Y]: [具体问题] - [为什么对实现重要]

    **建议（建议性，不阻止批准）：**
    - [改进建议]
```

**审查者返回：** 状态、问题（如有）、建议
