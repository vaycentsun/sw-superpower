## 验证检查清单

### 验证优先级

时间压力下不可平等对待所有检查项。优先级如下：

| 优先级 | 检查项 | 说明 |
|--------|--------|------|
| **不可跳过** | 功能验证、回归验证 | 核心功能不可用 = 不能标记完成 |
| **强烈建议** | 边界情况、错误处理 | 生产环境故障的主要来源 |
| **条件允许** | 性能、安全、兼容性 | 如项目有明确要求则必须验证 |

### 功能验证

- [ ] **主要功能** - 核心功能正常工作
- [ ] **次要功能** - 次要功能正常工作
- [ ] **用户流程** - 完整用户流程可执行
- [ ] **数据流** - 数据正确流动

### 边界情况

- [ ] **空输入** - 处理空值/空字符串
- [ ] **最大/最小值** - 处理极限值
- [ ] **特殊字符** - 处理特殊字符
- [ ] **大数据量** - 处理大量数据
- [ ] **并发** - 处理并发情况（如适用）

### 错误处理

- [ ] **无效输入** - 拒绝无效输入
- [ ] **异常情况** - 优雅处理异常
- [ ] **错误消息** - 错误消息清晰有用
- [ ] **恢复** - 能从错误中恢复

### 集成验证

- [ ] **依赖服务** - 依赖服务正常交互
- [ ] **数据库** - 数据库操作正确
- [ ] **API** - API 调用正确
- [ ] **文件系统** - 文件操作正确

### 非功能验证

- [ ] **性能** - 响应时间可接受
- [ ] **安全性** - 无安全漏洞
- [ ] **兼容性** - 目标环境兼容
- [ ] **资源使用** - 资源使用合理

### 回归验证（所有变更类型）

- [ ] **原有功能** - 未破坏现有功能（运行相关模块全部测试）
- [ ] **相关模块** - 变更影响范围内的模块功能正常
- [ ] **集成链路** - 端到端流程未因本次变更中断

## 按任务类型验证

**执行顺序**：变更范围确认 → 通用检查清单（必做） → 按任务类型重点验证（补充）

通用检查清单覆盖所有任务的基础验证，以下按任务类型提供额外的重点验证项：

### Bug 修复

**核心问题：修复是否生效 + 是否引入回归**

- [ ] **修复确认** - 原 Bug 场景不再复现（有重现步骤的先执行重现）
- [ ] **回归测试** - 运行相关模块的全部测试，确认未破坏原有功能
- [ ] **边界加固** - 修复位置附近的边界情况是否已覆盖
- [ ] **根因验证** - 确认修复针对的是根本原因，不是症状掩饰

### 新功能开发

**核心问题：功能完整 + 与需求一致**

- [ ] **功能完整** - 对照 Spec 逐条核对，所有计划行为已实现
- [ ] **集成验证** - 新功能与现有系统的交互正常
- [ ] **边界覆盖** - 空值、极限值、异常输入均有处理
- [ ] **验收测试** - 对照原始 Spec 的验收标准，逐项确认

### 重构

**核心问题：行为一致 + 无功能丢失**

- [ ] **行为一致性** - 输入输出与重构前完全一致（对比关键用例结果）
- [ ] **性能基准** - 关键路径性能无退化（如有基准测试则对比）
- [ ] **无功能丢失** - 所有原有功能仍正常工作
- [ ] **接口兼容** - 公开接口签名未发生破坏性变更（除非这是重构目标）

### 配置变更

**核心问题：环境可用 + 回滚可行**

- [ ] **环境验证** - 配置在目标环境生效，服务正常启动
- [ ] **依赖兼容** - 变更未引入版本冲突或依赖缺失
- [ ] **回滚方案** - 知道如何回滚此配置变更
- [ ] **默认值安全** - 未显式配置时的默认行为可接受

### 文档更新

**核心问题：内容准确 + 格式可用**

- [ ] **内容准确** - 文档描述与代码行为一致
- [ ] **链接可访问** - 所有超链接、交叉引用有效
- [ ] **格式正确** - 渲染后无格式错误（Markdown/HTML 等）
- [ ] **示例可运行** - 代码示例经过验证，可直接复制执行
